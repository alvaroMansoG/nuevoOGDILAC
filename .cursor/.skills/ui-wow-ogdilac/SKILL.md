# UI Wow Skill for OGDILAC

## Mission
Create a visually striking, premium, corporate, modern interface with strong art direction, while keeping clarity and information density high.

This project should feel:
- editorial
- institutional
- modern
- elegant
- high-trust
- visually memorable

It must NOT feel:
- generic SaaS template
- childish
- over-decorated
- startup toy
- dark dribbble clone
- overloaded with effects

## Core design intent
The visual target is a strong corporate portal inspired by:
- the Framer wireframe provided by the user
- a BID/IADB-style institutional digital aesthetic
- a polished, premium data product

The homepage must create a “wow” effect through:
- better composition
- stronger hierarchy
- more intentional use of scale
- richer surfaces
- controlled color contrast
- subtle depth
- elegant microinteractions
- premium spacing rhythm

## Non-negotiables
- Preserve the canonical country page logic and information architecture.
- Do not change the order of sections or charts without permission.
- Do not redesign the country page freely from scratch.
- Prefer visual enhancement over structural reinvention.
- Keep the interface highly readable and data-friendly.

## Art direction rules
- Use fewer but better visual ideas.
- Increase contrast between sections.
- Avoid large empty gray areas with no purpose.
- Give the hero more dramatic presence.
- Make featured cards feel more premium and less flat.
- Use asymmetry and composition to create visual interest.
- Create a stronger “editorial portal” feeling.
- Use subtle gradients, layered backgrounds, glow-free depth, refined borders, and soft shadows.
- Use stronger typography hierarchy.
- Use microinteractions to signal quality, not to distract.

## Preferred visual treatments
- Deep institutional blues and refined secondary tones
- Premium neutral backgrounds, not dull flat gray
- Large confident headlines
- More generous vertical rhythm
- Rounded cards with refined shadows
- Accent color used sparingly
- Section transitions that feel intentional
- Visual anchors in every viewport section

## Microinteraction rules
- Add hover lift on cards
- Add subtle transitions on navigation and buttons
- Add reveal-on-scroll where appropriate
- Add active/hover/focus states with clear polish
- Keep transitions subtle and fast
- No excessive motion
- No parallax unless explicitly requested

## Anti-patterns
Avoid:
- huge undifferentiated white/gray blocks
- weak hero sections
- flat card grids with no depth
- too many identical cards
- random bright gradients
- oversized shadows
- glassmorphism abuse
- trendy but low-trust UI patterns
- overcomplicated iconography

## Homepage-specific guidance
For the portal homepage:
- Make the hero visually stronger and more iconic
- Improve featured content cards so they feel premium
- Give countries/indicators/tools/library blocks stronger identity
- Use richer section backgrounds or separators
- Improve footer quality and structure
- Make “Próximamente” states elegant, not dull
- Make the whole page feel intentionally designed, not just assembled

## Output expectations
When working on UI:
1. First explain the proposed visual direction briefly.
2. Then implement.
3. Favor changes to spacing, hierarchy, composition, and surfaces before changing content structure.
4. If several visual directions are plausible, produce 2–3 alternatives.

## Review checklist
Before finishing, verify:
- Does this look more premium?
- Is the hero stronger?
- Are cards more memorable?
- Is the page less flat?
- Is the corporate feel stronger?
- Is readability still excellent?
- Does the UI feel designed, not generated?
